STF AI Diagnosis — RAG Ingestion Pipeline

MWS150-A Service Manual → Weaviate Vector Store  |  All models run locally via Ollama

1
CLI Entry & Schema Init
ingest.py • schema.py
  • CLI parses flags: --dir, --chunk-size 500, --overlap 50, --force-recreate, --enable-ocr, --enable-page-render, --enable-translation
  • Creates/recreates KnowledgeChunk collection in Weaviate (8 properties + vector)
  • Discovers *.pdf, *.txt, *.md files in the target directory
  • Detects source_type from filename: "manual" | "log" | "other"
2
PDF Structure Extraction
pdf_parser.py • parser.py
  • Font-based heading detection: computes body_size (most common font), then L1 heading ≥ 1.5× body, L2 ≥ 1.25× body
  • Splits document into Section objects: title, level, body, vehicle_model, dtc_codes
  • Extracts DTC codes via regex [PBCU]\d{4}  |  Vehicle model via pattern matching (STF-###, MWS-##-A, etc.)
  • Skips page numbers (^\d{1,2}-\d{1,3}$) and EAS/EWA/ECA codes
MWS150-A → 409 sections extracted
3
Image Enrichment (per page)
ocr.py • vision.py • pdf_parser.py
  • Three parallel enrichment tracks run for each PDF page (when flags are enabled):
OCR --enable-ocr
  • • EasyOCR (zh_tra + en)
  • • Extracts: part numbers, torque values, dimensions
  • • Dedup: skip if ≥80% overlap with text layer
  • • Inserts [OCR, Page N] markers
Vision llava
  • • Images ≥ 50x50 px, ≥ 5 KB
  • • Resize to max 1024 px longest side
  • • Describe diagrams, connectors, labels
  • • Inserts [Image N, Page M] markers
Full Page Render --enable-page-render
  • • Renders page at 150 DPI (~0.8 MB)
  • • Runs both OCR + Vision on full page
  • • Captures table layouts, mixed content
  • • Inserts [Full Page, Page N] markers
4
Chinese → English Translation
translator.py • cjk_utils.py
  • qwen3.5:9b   Sequential translation of each section via Ollama /api/generate
  • CJK detection: has_cjk() checks Unicode ranges U+2E80..U+9FFF, U+F900..U+FAFF, U+20000..U+2FA1F
  • Splits body on image markers → translates only CJK parts → preserves [Image N, Page M] blocks as-is
  • /no_think prefix disables qwen3 thinking mode; regex strips leaked <think> tags
Min CJK chars4
Max input length8,000 chars
Timeout120 s
Fallbackoriginal text
5
Section-Aware Chunking
chunker.py
  • Hierarchy: paragraphs → sentences → words (jieba for CJK, whitespace for English)
  • Normalize: insert \n\n after CJK sentence-end punctuation (。！？)
  • Image marker blocks treated as atomic units (never split mid-description)
  • Overlap: last 50 chars carried to next chunk, aligned to word/punctuation boundaries
Chunk size500 chars
Overlap50 chars
CJK segmenterjieba
OutputChunkedSection
6
Embedding Generation
embedding.py
  • nomic-embed-text   Each chunk → Ollama /api/embed → 768-dimensional float vector
  • Connection pooled via singleton httpx.AsyncClient
  • Empty text or network errors → returns [], chunk is skipped
Modelnomic-embed-text
Dimensions768
Timeout30 s
VRAM~261 MB
7
Dedup & Insert into Weaviate
ingest.py • schema.py
  • Checksum: SHA-256 of "{doc_id}:{section_title}:{chunk_text}"
  • Query Weaviate by checksum → skip if already exists (idempotent re-ingestion)
  • Insert: chunk text + 768-dim vector + all metadata properties
text: TEXT // chunk body (WHITESPACE tokenization)
doc_id: TEXT // e.g. "MWS150-A"
source_type: TEXT // "manual" | "log" | "other"
section_title: TEXT // heading (WORD tokenization)
vehicle_model: TEXT // "MWS-150-A"
chunk_index: INT // sequential index
checksum: TEXT // SHA-256 for dedup
metadata_json: TEXT // {"dtc_codes":[], "has_image":false}
_vector: FLOAT[768] // nomic-embed-text embedding
→
Retrieval (query time)
retrieve.py
  • User query → nomic-embed-text embedding → nearVector search on KnowledgeChunk
  • Returns top-K results with score (1.0 - distance), doc_id, section_title, metadata
  • Results feed into diagnostic API → qwen3.5:9b for final diagnostic response
